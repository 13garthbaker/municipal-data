List Facts


  
Used to return individual entries from the cube in a non-aggregated form. Supports filters (cut), a set of fields to return and a sort (field_name:direction), as well as page and page_size.


  
GET /cubes/<name>/facts


  
Cut (filter)


  
Use the parameter cut with values key:value to restrict values to a subset of the cube. This is called cutting the cube into cells. Separate multiple cuts with vertical bar e.g. cut=item.code:"3010"|municipality.code:"BUF"

  
String values must be quoted with double quotes


  
GET /cubes/<name>/facts?cut=item.code:"3010"|municipality.code:"BUF"


  
Sort (order)


  
Use the parameter order to sort the returned values. Separate multiple sorting dimensions with , e.g. order=financial_year_end.year:desc|amount_type.code:asc

  
The results will be primarily sorted according to the first argument; within that, it is sorted according to the second argument, etc. That means it's equivalent to a stable sort performed in reverse order of the arguments.



  
List members


  
Used to return the distinct set of values for a given dimension, e.g. all the suppliers mentioned in a procurement dataset. Supports filters (cut),  sort, as well as page and page_size.


  
GET /cubes/<name>/members/<member_name>


  
Aggregate


  
The main endpoint for generating aggregate views of the data. Supports specifying the aggregates to include, the drilldowns to aggregate by, a set of filters (cut), a and a sort (field_name:direction), as well as page and page_size.


  
GET /cubes/<name>/aggregate
